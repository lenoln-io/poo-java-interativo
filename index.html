Estruturas de Controle: Repetição

As estruturas de repetição, ou laços, são essenciais para executar um bloco de código várias vezes. Java oferece diferentes tipos de laços, cada um adequado para um cenário específico. Explore-os interativamente abaixo.

Quando usar for?

Ideal para quando você sabe exatamente quantas vezes quer que o laço seja executado. É comum para iterar sobre arrays ou executar uma ação um número fixo de vezes.

for (int i = 0; i < 5; i++) {
    System.out.println("O valor de i é: " + i);
}

Quando usar while?

                    
Perfeito para situações onde o laço deve continuar enquanto uma condição for verdadeira. O número de iterações não precisa ser conhecido de antemão. O bloco pode não ser executado nenhuma vez se a condição for inicialmente falsa.

                     

                        
int contador = 0;
while (contador < 3) {
    System.out.println("Contador: " + contador);
    contador++;
}

Quando usar do-while?

                    
Similar ao while, mas com uma diferença crucial: o bloco de código é executado pelo menos uma vez, pois a condição é verificada no final da iteração. Útil para menus de opções ou quando uma ação inicial é sempre necessária.

                     

                        
int j = 0;
do {
    System.out.println("O valor de j é: " + j);
    j++;
} while (j < 3);

Conceitos de Programação Orientada a Objetos

            

                A POO organiza o código em torno de "objetos", que unem dados (atributos) e comportamentos (métodos). Isso torna o software mais organizado, reutilizável e fácil de manter.
            

            
            

                 

                    
Classe

                    
Um modelo ou "planta baixa" para criar objetos. Define os atributos e métodos que seus objetos terão.


                

                 

                    
Objeto

                    
Uma instância de uma classe. É a "construção" real feita a partir da planta baixa, com seus próprios dados.


                

                

                    
Encapsulamento

                    
Ocultar os detalhes internos de um objeto, protegendo seus dados e expondo apenas o necessário através de métodos.


                

                

                    
Herança

                    
Permite que uma classe (filha) herde atributos e métodos de outra classe (pai), promovendo a reutilização de código.

   
                

            


            

                
Fábrica de Objetos: Classe vs. Objeto na Prática

                
Use a "planta" da classe Carro para criar objetos. Cada objeto criado será uma instância independente, com seus próprios atributos.

                

                    

                        
Planta Baixa (Classe)

                        
public class Carro {
    String marca;
    String modelo;
    int ano;
    String cor;

    void ligar() {
        System.out.println(modelo + " ligado!");
    }
}

Simulação Prática de POO

            

                Vamos analisar o código de exemplo do relatório que usa uma classe Pessoa. Clique em "Simular" para ver a criação de objetos e a chamada de métodos acontecerem passo a passo.
            

            

                

                    

                        
Código Fonte

                        

                            
                                
public class Pessoa {
    String nome;
    int idade;

    void aniversario() {
        idade++;
    }

    void apresentar() {
        System.out.println(
          "Meu nome é " + nome + 
          " e tenho " + idade + " anos."
        );
    }
}

                            
                          
public static void main(String[] args) {
  Pessoa p1 = new Pessoa();
  p1.nome = "Maria";
  p1.idade = 25;
  Pessoa p2 = new Pessoa();
  p2.nome = "João";
  p2.idade = 30;
  p1.apresentar();
  p2.apresentar();
  p1.aniversario();
  p1.apresentar();
}
